Sinkronisasi Google Sheets

Publish sheet LOG VISIT dan TRANSAKSI ke web sebagai CSV (File → Share → Publish to web), lalu tempel link-nya di sini. Header kolom harus sama persis dengan file asal.

Sheet baru khusus master outlet (kolom: NAMA OUTLET, WILAYAH, TIPE OUTLET, STATUS OUTLET, NO. KONTAK, PIC — kolom KEY dibuat otomatis) — dengan ini, perubahan di "Kelola Outlet" ikut tersimpan ke Google Sheets, bukan cuma di browser ini.

Sheet baru khusus jadwal (kolom: KEY, MINGGU, SALES, HARI, TANGGAL, SLOT, OUTLET, STATUS_HARI, ALASAN_LIBUR, FOLLOWUP, KETERANGAN_FU, RESOLVED) — dipakai tombol "Muat/Sync ke Sheets" di Jadwal Mingguan & tab Follow Up.

Sheet baru khusus pricelist (kolom: KEY, KATEGORI, KEMASAN, PRODUK, SIZE, QTY_KARTON, A_DBP, A_WBP, A_RBP, B_DBP, B_WBP, B_RBP) — dipakai tombol "Muat/Sync ke Sheets" di Kelola Pricelist.

Sheet baru khusus catatan follow up mandiri (kolom: KEY, TANGGAL, SALES, OUTLET, KETERANGAN, RESOLVED) — dipakai untuk catatan yang ditambah langsung di tab Follow Up, tanpa lewat kunjungan/jadwal.

Sheet baru khusus target sales (kolom: KEY, SALES, PERIODE, TARGET_OMZET, TARGET_CALL, TARGET_EC, TARGET_NOO) — dipakai tombol "Muat/Sync ke Sheets" di Target Sales.

Sheet baru khusus Pengajuan Biaya Marketing (kolom: KEY, PIC, KETERANGAN, PERIODE_MULAI, PERIODE_SELESAI, TARGET_PENJUALAN, PROMOS_JSON, GIMMICKS_JSON, TTD_DIBUAT_NAMA, TTD_DIBUAT_JABATAN, TTD_TAHU_NAMA, TTD_TAHU_JABATAN, TTD_SETUJU_NAMA, TTD_SETUJU_JABATAN, OUTLETS, STATUS, TANGGAL_DIBUAT, EVAL_OMZET_AKTUAL, EVAL_TRANSAKSI, EVAL_PENGUNJUNG, EVAL_APA_BERHASIL, EVAL_PERLU_DIPERBAIKI, EVAL_RENCANA, EVAL_TANGGAL) — dipakai fitur Kalkulator Promo.

Memakai data bawaan (snapshot saat file dibuat).
Tujuan Simpan Data Baru

URL Web App Google Apps Script yang menerima kunjungan baru dari form "Input Kunjungan" (ditambahkan ke sheet LOG VISIT), dan juga perubahan dari "Kelola Outlet" (disimpan ke sheet MASTER OUTLET). Lihat panduan setup di bawah tombol Input Kunjungan.

Auto Sync & Muat Otomatis

Otomatis kirim (Outlet, Jadwal Mingguan, Pricelist) lalu tarik ulang semua data dari Sheets secara berkala — supaya perubahan di device lain ikut muncul tanpa klik manual. Catatan: publish-to-web Google butuh 1–5 menit untuk update, jadi interval terlalu singkat tidak menjamin data langsung terbaru.

Input Kunjungan Baru

Belum terhubung ke Google Sheets. Buka ⚙ Sumber Data di pojok kanan atas, isi bagian "Tujuan Simpan Kunjungan Baru" dengan URL Apps Script Web App.

Setup Apps Script — Kunjungan, Master Outlet & Jadwal Mingguan

Versi ini pakai sistem KEY ID: setiap outlet punya satu ID tetap (OUT-0001, OUT-0002, dst) di MASTER OUTLET, dan setiap baris Kunjungan/Transaksi/Jadwal/Follow Up ikut menyimpan OUTLET_KEY-nya — jadi satu outlet punya satu riwayat lengkap yang akurat, tidak tergantung ejaan nama. Kolom KEY/OUTLET_KEY dibuat & diisi otomatis oleh script di bawah, tidak perlu ditambah manual.

  1. Buat sheet baru bernama persis MASTER OUTLET di spreadsheet Anda, baris pertama diisi header: NAMA OUTLET, WILAYAH, TIPE OUTLET, STATUS OUTLET, NO. KONTAK, PIC (boleh tambah kolom lain, tidak masalah — kolom KEY akan ditambahkan otomatis).
  2. Di sheet LOG VISIT Anda, pastikan ada kolom NOO (isi TRUE/FALSE — dicentang otomatis dari checkbox "NOO" di form Input Kunjungan). Nilai kolom Status Outlet sekarang "Existing" atau "Outlet Baru" (dulu "NOO" — outlet lama bertanda "NOO" tetap terbaca sebagai "Outlet Baru" di dashboard).
  3. Buat juga sheet baru bernama persis JADWAL MINGGUAN, header: KEY, MINGGU, SALES, HARI, TANGGAL, SLOT, OUTLET, STATUS_HARI, ALASAN_LIBUR, FOLLOWUP, KETERANGAN_FU, RESOLVED (kolom OUTLET_KEY ditambahkan otomatis).
  4. Buat juga sheet baru bernama persis PRICELIST, header: KEY, KATEGORI, KEMASAN, PRODUK, SIZE, QTY_KARTON, A_DBP, A_WBP, A_RBP, B_DBP, B_WBP, B_RBP.
  5. Buat juga sheet baru bernama persis CATATAN FOLLOWUP, header: KEY, TANGGAL, SALES, OUTLET, KETERANGAN, RESOLVED (kolom OUTLET_KEY ditambahkan otomatis).
  6. Buat juga sheet baru bernama persis TARGET, header: KEY, SALES, PERIODE, TARGET_OMZET, TARGET_CALL, TARGET_EC, TARGET_NOO — dipakai fitur Target Sales.
  7. Buat juga sheet baru bernama persis PENGAJUAN, header: KEY, PIC, KETERANGAN, PERIODE_MULAI, PERIODE_SELESAI, TARGET_PENJUALAN, PROMOS_JSON, GIMMICKS_JSON, TTD_DIBUAT_NAMA, TTD_DIBUAT_JABATAN, TTD_TAHU_NAMA, TTD_TAHU_JABATAN, TTD_SETUJU_NAMA, TTD_SETUJU_JABATAN, SERTAKAN_TTD, OUTLETS, STATUS, TANGGAL_DIBUAT, EVAL_OMZET_AKTUAL, EVAL_TRANSAKSI, EVAL_PENGUNJUNG, EVAL_APA_BERHASIL, EVAL_PERLU_DIPERBAIKI, EVAL_RENCANA, EVAL_TANGGAL — dipakai fitur Kalkulator Promo (Pengajuan Biaya Marketing). Kolom PROMOS_JSON dan GIMMICKS_JSON isinya teks JSON (daftar promo/gimmick di pengajuan itu, termasuk evaluasi per-promonya) — jangan diedit manual di sheet, cukup dibiarkan apa adanya. SERTAKAN_TTD isinya TRUE/FALSE — menentukan apakah blok Tanda Tangan & Persetujuan ikut muncul di PDF pengajuan itu.
  8. Isi TTD_DIBUAT_NAMA dst di atas cuma dipakai kalau Anda tidak menyimpan "Template Tanda Tangan" di dashboard (Kalkulator Promo → tombol "Simpan sebagai Template") — kalau sudah ada template, field itu otomatis terisi sendiri tiap bikin pengajuan baru, jadi tidak perlu ketik ulang.
  9. Buka Extensions → Apps Script dari spreadsheet Anda.
  10. Hapus isi default, lalu tempel kode di bawah ini.
  11. (Opsional) Ganti nilai SECRET dengan kode rahasia Anda sendiri, lalu isi kode yang sama di panel "Sumber Data" pada dashboard.
  12. Klik Deploy → New deployment → pilih tipe Web app.
  13. Execute as: Me. Who has access: Anyone. Klik Deploy, lalu izinkan (Authorize) saat diminta.
  14. Salin URL Web App yang muncul (diakhiri /exec), tempel di panel "Sumber Data" → "URL Apps Script (Web App)". URL yang sama ini dipakai untuk Input Kunjungan, Kelola Outlet, Jadwal Mingguan, Pricelist, Target Sales, dan Kalkulator Promo.
  15. Publish sheet MASTER OUTLET, JADWAL MINGGUAN, PRICELIST, CATATAN FOLLOWUP, TARGET, dan PENGAJUAN juga ke web sebagai CSV (sama seperti LOG VISIT/TRANSAKSI), lalu tempel link-nya masing-masing di panel Sumber Data supaya dashboard bisa membaca data itu juga (bukan cuma menulis).
  16. Kalau sheet Anda sudah punya data lama (sebelum pakai versi KEY ID ini): di editor Apps Script, pilih fungsi migrateAddOutletKeys dari dropdown atas, lalu klik Run — sekali saja. Ini mengisi OUTLET_KEY/KEY untuk semua baris lama berdasar nama outlet yang sudah ada.

Kelola Outlet

        
      

      

        
        

        
        
        
      

      

      

        Daftar ini menentukan pilihan outlet di form Input Kunjungan ke depannya (biar penulisan konsisten).
        Ubah nama jadi sama persis untuk menggabungkan dua baris yang sebetulnya outlet yang sama.
        Isi kolom Wilayah (kecamatan/area) untuk membantu mengelompokkan outlet yang berdekatan saat membagi rute sales — lalu urutkan berdasarkan wilayah untuk mempermudah pengisian massal.
        Perubahan di sini otomatis tersimpan di browser ini, dan kalau URL Apps Script sudah disetel di Sumber Data, ikut tersinkron ke sheet MASTER OUTLET di Google Sheets Anda.
        Saat mengetik Nama Outlet baru dengan kolom Wilayah masih kosong, dashboard otomatis mencari kecamatannya lewat Google Maps (butuh URL Apps Script tersambung & Maps Service aktif) — tetap bisa diedit manual kalau hasilnya kurang tepat.
